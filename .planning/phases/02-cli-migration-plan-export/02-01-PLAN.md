**Create `src/main/kotlin/nesski/de/cli/WahooCli.kt`:**
- Class `WahooCli` extends `CliktCommand` with:
  - `name = "wahoo-cli"`
  - `help = "Fetch Wahoo SYSTM training plans and display them"`
- Options (all using Clikt `option()`):
  - `--range` / `-r`: String?, help = "Time range shorthand: now, 1w, 2w, 1m, 2m"
  - `--from`: String?, help = "Start date (YYYY-MM-DD)"
  - `--to`: String?, help = "End date (YYYY-MM-DD)"
  - `--config` / `-c`: String, help = "Config file path", default = "~/.config/wahoo-cli/config" (expand `~` to `System.getProperty("user.home")` at runtime, not in default)
- `override fun run()`:
  1. Resolve config path (expand `~` in configPath to user.home)
  2. Load config via `AppConfig.load(resolvedConfigPath)`
  3. Resolve credentials: env vars (`SYSTM_USER`, `SYSTM_PASSWORD`) override config values. If neither has credentials, call `promptForCredentials()` which uses Clikt's `prompt()` for username and `prompt(hideInput=true)` for password, then offers to save via `confirm()`
  4. Parse date range via `parseDateRange(range, from, to)` — wrap in try/catch, on error use Clikt `throw UsageError(message)` for clean CLI error output
  5. For now (skeleton), echo the parsed config and date range: `echo("Fetching plans from ${dateRange.start} to ${dateRange.end}")`
  6. Add a `// TODO: Wire auth, fetch, display in Plan 02` comment where the full orchestration will go

**Create `src/main/kotlin/nesski/de/config/AppConfig.kt`:**
- Data classes (all `@Serializable`):
  - `AppConfig(credentials: CredentialsConfig, output: OutputConfig)` — all fields have defaults
  - `CredentialsConfig(username: String = "", password: String = "")`
  - `OutputConfig(@SerialName("ics_save_path") icsSavePath: String = ".")` — keep this for Phase 3 forward-compatibility
- Companion object on `AppConfig`:
  - `fun load(configPath: String): AppConfig` — if file exists, use `TomlFileReader.decodeFromFile<AppConfig>(serializer(), configPath)`, otherwise return `AppConfig()` (defaults). Catch exceptions and log a warning, returning defaults.
  - `fun resolveCredentials(config: AppConfig): Pair<String, String>` — env var `SYSTM_USER` overrides `config.credentials.username`, `SYSTM_PASSWORD` overrides `config.credentials.password`
- Note: SmtpConfig is NOT included yet — it belongs to Phase 3 (export). The TOML config structure is extensible so Phase 3 just adds a new `[smtp]` section.
- Use `com.akuleshov7.ktoml.file.TomlFileReader` for file reading
- Import `kotlinx.serialization.Serializable` and `kotlinx.serialization.SerialName`

**Create `src/main/kotlin/nesski/de/utils/DateRangeParser.kt`:**
- Data class `DateRange(val start: LocalDate, val end: LocalDate)`
- Function `parseDateRange(range: String?, from: String?, to: String?): DateRange`:
  1. If `range != null && (from != null || to != null)` → throw `IllegalArgumentException("Cannot use --range with --from/--to. Use one or the other.")`
  2. If `range != null` → parse shorthand: `"now"` → today..today, `"1w"` → today..today+1week, `"2w"` → today..today+2weeks, `"1m"` → today..today+1month, `"2m"` → today..today+2months. Unknown value → throw `IllegalArgumentException("Invalid range: $range. Valid values: now, 1w, 2w, 1m, 2m")`
  3. If `from != null || to != null` → parse ISO dates with `LocalDate.parse()`. If only `from`, end = from+2weeks. If only `to`, start = today. Enforce max 2 months via `ChronoUnit.DAYS.between() > 62` → throw error.
  4. Default (no range flags): today..today+2weeks (per user decision: default is next 2 weeks)
- Use `java.time.LocalDate`, `java.time.temporal.ChronoUnit`
