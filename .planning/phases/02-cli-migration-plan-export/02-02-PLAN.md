**Update `src/main/kotlin/nesski/de/cli/WahooCli.kt` — replace the skeleton `run()` with full orchestration:**

The `run()` method should implement this flow (all wrapped in `runBlocking` since Ktor client calls are suspend functions):

```
1. Load config (already in skeleton from Plan 01)
2. Resolve credentials (already in skeleton from Plan 01)
3. Parse date range (already in skeleton from Plan 01)
4. Authenticate with SYSTM API
5. Fetch plans for date range
6. Display plans in console
```

**Step 4 — Authenticate:**
- Create `SystmAuthService(wahooHttpClient, username, password)` using the HTTP client from `SystmGraphQLClient.kt`
- Call `authService.login()` in a try/catch
- On success: echo `"✓ Authenticated as ${username}"`, store token
- On failure: echo `"✗ Authentication failed: ${e.message}"`, call `throw ProgramResult(1)` (Clikt's way to exit with non-zero code)

**Step 5 — Fetch plans:**
- Create `SystmPlansService(wahooHttpClient)`
- Call `plansService.fetchPlans(token, dateRange.start, dateRange.end)` in a try/catch
- On `GraphQLException`: echo `"✗ API error: ${e.message}"`, exit 1
- On other exception: echo `"✗ Failed to fetch plans: ${e.message}"`, exit 1
- On success: store plans list

**Step 6 — Display plans in console (DISP-01, DISP-02, DISP-03):**
- Echo header: `"\nTraining Plans (${dateRange.start} to ${dateRange.end}):\n"`
- If plans is empty: echo `"No plans found for this date range."` and exit 0
- For each plan in plans:
  - Echo plan header: `"📋 ${plan.name} (${plan.status ?: "scheduled"})"`
  - For each workout in plan.workouts:
    - Echo: `"  ${workout.name} — ${workout.scheduledDate ?: "no date"} [${workout.type ?: "unknown"}] [${workout.status ?: "planned"}]"`
    - This satisfies PARSE-01 (name), PARSE-02 (date), PARSE-03 (type), PARSE-04 (status)
- Echo summary: `"\n${totalWorkouts} workout(s) across ${plans.size} plan(s)\n"`
