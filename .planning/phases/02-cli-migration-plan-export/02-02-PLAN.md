Create `src/main/kotlin/nesski/de/services/IcsGeneratorService.kt` with:

**Emoji mapping function `emojiFor(type: String?): String`:**
- `"yoga"` → `"🧘🏽‍♂️"` (per user decision — skin-toned variant)
- `"cycling"` → `"🚴"` (per user decision)
- `"strength"` → `"🏋🏾‍♀️"` (per user decision — skin-toned variant)
- `"running"` → `"🏃"`
- `"swimming"` → `"🏊"`
- `"mental"`, `"mental training"` → `"🧠"`
- `else` / `null` → `"🏋️"` (generic fitness fallback)
- Use `type?.lowercase()` for case-insensitive matching

**Class `IcsGeneratorService`:**
- Function `generateIcs(workouts: List<Workout>): String`:
  1. Create `ICalendar` instance, set PRODID to `"-//Wahoo CLI//SYSTM Plan Export//EN"`
  2. For each workout in the list:
     - Create `VTodo` instance
     - Set SUMMARY: `"${emojiFor(workout.type)} ${workout.name} (${workout.type ?: "Workout"})"` (per user decision format)
     - Set DUE: Parse `workout.scheduledDate` (String, ISO format) to `LocalDate`, then convert to `java.util.Date` using `Date.from(localDate.atStartOfDay(ZoneId.systemDefault()).toInstant())`. Use biweekly's `vtodo.setDateDue()` — IMPORTANT: use DATE-only (not DATE-TIME) for Apple Reminders compatibility as an all-day task
     - Set STATUS: If `workout.status?.lowercase() == "completed"` → `Status.completed()`, else → `Status.needsAction()`
     - Set UID: Use `workout.id` as the unique identifier (biweekly's `vtodo.setUid(workout.id)`)
     - DTSTAMP: biweekly sets this automatically — do not set manually
     - DESCRIPTION: Leave empty (per user decision — no description/notes)
     - Add vtodo to ical via `ical.addTodo(vtodo)`
  3. Serialize with `Biweekly.write(ical).go()` — this returns a UTF-8 String

- Function `saveToFile(icsContent: String, filePath: String)`:
  1. Create parent directory if it doesn't exist: `File(filePath).parentFile?.mkdirs()`
  2. Write icsContent to file with UTF-8 charset: `File(filePath).writeText(icsContent, Charsets.UTF_8)`

**Imports needed:**
- `biweekly.ICalendar`, `biweekly.Biweekly`, `biweekly.component.VTodo`, `biweekly.property.Status`
- `nesski.de.models.Workout`
- `java.time.LocalDate`, `java.time.ZoneId`
- `java.util.Date`
- `java.io.File`

**IMPORTANT Apple Reminders compatibility notes (from research):**
- Every VTODO must have: UID, DTSTAMP, DUE, STATUS, SUMMARY
- DUE should be a DATE (not DATE-TIME) for all-day task appearance
- biweekly handles CRLF line endings and 75-octet line folding automatically

Create `src/main/kotlin/nesski/de/services/EmailService.kt` with:

**Class `EmailService`:**

- Constructor takes `smtpConfig: SmtpConfig` (from `nesski.de.config.AppConfig`)

- Function `sendIcsEmail(recipientEmail: String, subject: String, body: String, icsContent: String, icsFilename: String)`:
  1. Build email using Simple Java Mail's `EmailBuilder.startingBlank()`:
     - `.from("Wahoo CLI", smtpConfig.user)` — sender is the SMTP user
     - `.to(recipientEmail)` — recipient
     - `.withSubject(subject)` — date-range based subject
     - `.withPlainText(body)` — plain text body with workout count
     - `.withAttachment(icsFilename, icsContent.toByteArray(Charsets.UTF_8), "text/calendar; charset=utf-8")` — .ics attachment with correct MIME type and charset for emoji support
     - `.buildEmail()`
  2. Build mailer using `MailerBuilder`:
     - `.withSMTPServer(smtpConfig.host, smtpConfig.port, smtpConfig.user, smtpConfig.pass)`
     - `.withTransportStrategy(TransportStrategy.SMTP_TLS)` — STARTTLS on port 587 (most common). Per user discretion on exact TLS handling.
     - `.buildMailer()`
  3. Call `mailer.sendMail(email)` — this is synchronous (blocking)

- Function `formatSubject(startDate: LocalDate, endDate: LocalDate): String`:
  - Format: `"Wahoo Training Plan: Mar 8 - Mar 22, 2026"` (per user decision)
  - Use `DateTimeFormatter.ofPattern("MMM d")` for start date, `DateTimeFormatter.ofPattern("MMM d, yyyy")` for end date
  - Return `"Wahoo Training Plan: ${start} - ${end}"`

- Function `formatBody(workoutCount: Int, startDate: LocalDate, endDate: LocalDate): String`:
  - Format: `"8 workouts from Mar 8-22. Open the attachment to import."` (per user decision)
  - Use same date formatters
  - Return `"$workoutCount workouts from ${start}-${endDay}. Open the attachment to import."`

**Imports needed:**
- `org.simplejavamail.api.mailer.config.TransportStrategy`
- `org.simplejavamail.email.EmailBuilder`
- `org.simplejavamail.mailer.MailerBuilder`
- `nesski.de.config.SmtpConfig`
- `java.time.LocalDate`, `java.time.format.DateTimeFormatter`

**Error handling:** Let exceptions from `sendMail()` propagate — the caller (WahooCli in Plan 03) will catch them and handle the fallback-to-disk logic.
