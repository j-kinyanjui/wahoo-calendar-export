**Update `src/main/kotlin/nesski/de/cli/WahooCli.kt` — replace the skeleton `run()` with full orchestration:**

The `run()` method should implement this flow (all wrapped in `runBlocking` since Ktor client calls are suspend functions):

```
1. Load config (already in skeleton from Plan 01)
2. Resolve credentials (already in skeleton from Plan 01)
3. Parse date range (already in skeleton from Plan 01)
4. Authenticate with SYSTM API
5. Fetch plans for date range
6. Display plans in console
7. Flatten workouts from plans
8. Generate .ics file
9. Email .ics or save to disk
```

**Step 4 — Authenticate:**
- Create `SystmAuthService(wahooHttpClient, username, password)` using the HTTP client from `SystmGraphQLClient.kt`
- Call `authService.login()` in a try/catch
- On success: echo `"✓ Authenticated as ${username}"`, store token
- On failure: echo `"✗ Authentication failed: ${e.message}"`, call `throw ProgramResult(1)` (Clikt's way to exit with non-zero code)

**Step 5 — Fetch plans:**
- Create `SystmPlansService(wahooHttpClient)`
- Call `plansService.fetchPlans(token, dateRange.start, dateRange.end)` in a try/catch
- On `GraphQLException`: echo `"✗ API error: ${e.message}"`, exit 1
- On other exception: echo `"✗ Failed to fetch plans: ${e.message}"`, exit 1
- On success: store plans list

**Step 6 — Display plans in console (DISP-01, DISP-02, DISP-03):**
- Echo header: `"\nTraining Plans (${dateRange.start} to ${dateRange.end}):\n"`
- If plans is empty: echo `"No plans found for this date range."` and exit 0 (nothing to export)
- For each plan in plans:
  - Echo plan header: `"📋 ${plan.name} (${plan.status ?: "scheduled"})"`
  - For each workout in plan.workouts:
    - Echo: `"  ${emojiFor(workout.type)} ${workout.name} — ${workout.scheduledDate ?: "no date"} [${workout.status ?: "planned"}]"`
    - This satisfies PARSE-01 (name), PARSE-02 (date), PARSE-03 (type via emoji), PARSE-04 (status)
- Echo summary: `"\n${totalWorkouts} workout(s) across ${plans.size} plan(s)\n"`

**Step 7 — Flatten workouts:**
- `val allWorkouts = plans.flatMap { it.workouts }`
- If empty, echo `"No workouts to export."` and exit 0

**Step 8 — Generate .ics:**
- Create `IcsGeneratorService()`
- Call `val icsContent = icsGenerator.generateIcs(allWorkouts)`
- Determine filename: `"wahoo-plan-${dateRange.start}-to-${dateRange.end}.ics"` (per OpenCode's discretion on naming)

**Step 9 — Email or save:**
- If `email` option is provided:
  - Resolve SMTP config via `AppConfig.resolveSmtpConfig(config)`
  - Validate SMTP config has required fields (host, user, pass non-empty). If missing: echo `"✗ SMTP not configured. Set SMTP_HOST, SMTP_USER, SMTP_PASS env vars or add [smtp] section to config."`, save .ics to disk instead, exit 1
  - Create `EmailService(smtpConfig)`
  - Format subject via `emailService.formatSubject(dateRange.start, dateRange.end)`
  - Format body via `emailService.formatBody(allWorkouts.size, dateRange.start, dateRange.end)`
  - Try `emailService.sendIcsEmail(email, subject, body, icsContent, icsFilename)`
  - On success: echo `"✓ .ics file emailed to ${email}"`
  - On failure: echo `"✗ Email failed: ${e.message}"`. Save .ics to disk as fallback (using `config.output.icsSavePath`): `icsGenerator.saveToFile(icsContent, "${config.output.icsSavePath}/$icsFilename")`. Echo `"  .ics saved to ${savePath} instead. You can import it manually."`
- If `email` option is NOT provided:
  - Always save to disk: `val savePath = "${config.output.icsSavePath}/$icsFilename"`. Call `icsGenerator.saveToFile(icsContent, savePath)`. Echo `"✓ .ics file saved to ${savePath}"`
