# Plan 05-01: GitHub Actions CI Workflow with Gradle Caching

## Objective

Create a GitHub Actions CI workflow that builds and tests the project on every push to main and on PRs targeting main. Configure Gradle dependency caching for faster subsequent runs, enforce ktfmt formatting checks in CI, and configure Dependabot for automated Gradle dependency updates.

<tasks>
  <task id="ci-workflow">
    Create `.github/workflows/ci.yml` GitHub Actions workflow:
    - Name: "CI"
    - Triggers: `push` to `main` branch, `pull_request` targeting `main` branch
    - Single job: `build`
    - Runner: `ubuntu-latest`
    - Steps:
      1. Checkout code (`actions/checkout@v4`)
      2. Set up JDK 17 (`actions/setup-java@v4` with `distribution: temurin`)
      3. Set up Gradle with caching (`gradle/actions/setup-gradle@v4` — handles dependency caching automatically via CI-03)
      4. Make gradlew executable (`chmod +x ./gradlew`)
      5. Run build (`./gradlew build` — compiles, runs all tests, packages; satisfies CI-01 and CI-02)
      6. Run ktfmt check (`./gradlew ktfmtCheck` — fail if code isn't formatted per project style)
    - Gradle wrapper validation is handled by `gradle/actions/setup-gradle@v4` by default
    - Test results visible in workflow logs (console output, no artifact upload per CONTEXT decisions)
  </task>
  <task id="dependabot-config">
    Update `.github/dependabot.yml` to configure Gradle dependency updates:
    - Set `package-ecosystem` to `"gradle"`
    - Keep `directory: "/"` and `schedule.interval: "weekly"`
    - Add `open-pull-requests-limit: 10` for reasonable PR volume
  </task>
  <task id="branch-protection">
    Configure branch protection rules for `main` branch via GitHub CLI (`gh`):
    - Require status checks to pass before merging (enforce the "build" job from CI workflow)
    - Block direct pushes to main — all changes must go through PRs
    - No review requirement (single-contributor project — self-merge after CI passes)
    - Command: `gh api repos/{owner}/{repo}/branches/main/protection -X PUT` with appropriate JSON payload
    - Verify: attempt a direct push to main is blocked; PRs require CI status check
  </task>
</tasks>

## Verification
- `./gradlew build` succeeds locally before pushing
- Pushing a commit to `main` triggers the CI workflow (CI-01)
- Opening a PR targeting `main` triggers the CI workflow (CI-01)
- All tests execute during CI; a failing test causes workflow failure with visible error output (CI-02)
- Second workflow run is faster than first due to Gradle dependency caching (CI-03)
- ktfmt formatting check runs in CI and would fail on unformatted code
- Dependabot creates PRs for outdated Gradle dependencies weekly
- Branch protection blocks direct pushes to main
- PRs to main require CI status check to pass before merge

## Must Haves
- GitHub Actions workflow triggers on push to main and PRs to main
- All tests run during CI with visible pass/fail status
- Gradle dependencies cached between workflow runs
- ktfmt formatting enforced in CI
- Dependabot configured for Gradle ecosystem
- Branch protection rules enforce CI-gated merges on main
