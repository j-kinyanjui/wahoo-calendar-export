# Plan 04-01: Migrate VTODO to VEVENT using ical4j

## Objective

Replace hand-rolled VTODO ICS generation with ical4j-based VEVENT all-day calendar events. Workouts become universally compatible calendar placeholders that users can drag to their preferred time.

<tasks>
  <task id="ical4j-dep">
    Add ical4j dependency to build.gradle.kts. Verify it resolves and compiles.
  </task>
  <task id="ics-rewrite">
    Rewrite IcsBuilder to generate VEVENT all-day placeholder events using ical4j API. Replace VTODO with VEVENT (DATE-only DTSTART/DTEND). Include UID with @wahoo suffix, DTSTAMP, SUMMARY with duration hint, DESCRIPTION with drag instruction, STATUS:CONFIRMED, TRANSP:TRANSPARENT, CALSCALE:GREGORIAN. Remove VTODO-specific code (DUE, STATUS mapping). Keep SportEmoji integration and skip logic (rest days, missing dates). Output via ical4j CalendarOutputter for RFC 5545 compliance.
  </task>
  <task id="test-migration">
    Update all ICS tests: replace VTODO assertions with VEVENT assertions. Verify DTSTART;VALUE=DATE, DTEND;VALUE=DATE, no DUE property, STATUS:CONFIRMED, TRANSP:TRANSPARENT, CALSCALE:GREGORIAN. Update SummaryFormattingTest for new format with duration hint. Update EmailServiceTest sample ICS content. Update IcsExportIntegrationTest if affected.
  </task>
  <task id="email-update">
    Update EmailService body text to reference calendar events instead of Apple Reminders. Remove VTODO/reminder language from all user-facing strings.
  </task>
</tasks>

## Verification
- All tests pass with `./gradlew test`
- Generated .ics contains VEVENT (not VTODO)
- Generated .ics contains CALSCALE:GREGORIAN
- Generated .ics contains DTSTART;VALUE=DATE and DTEND;VALUE=DATE
- Generated .ics contains STATUS:CONFIRMED and TRANSP:TRANSPARENT
- No VTODO, DUE, or NEEDS-ACTION in output

## Must Haves
- Valid cross-calendar-compatible .ics output (Apple, Google, Outlook, Yahoo)
- ical4j used for ICS generation (not hand-rolled strings)
- All existing tests updated and passing
