## Phases

### Phase 5: CI Pipeline

**Goal:** Automated build and test on every push and PR to main, with Gradle dependency caching.

**Status:** Complete (2026-03-14) — 1/1 plan executed

**Success Criteria:**
1. [x] Pushing a commit to main triggers a GitHub Actions workflow that builds the project
2. [x] Opening a PR to main triggers the same workflow (configured)
3. [x] All tests run during CI; a failing test causes the workflow to fail with visible error
4. [x] Subsequent workflow runs use cached Gradle dependencies (faster than first run)

**Plans:**
- [x] **05-01** — GitHub Actions CI Workflow with Gradle Caching

**Key decisions:**
- `gradle/actions/setup-gradle@v4` handles caching automatically
- `spotlessCheck` (not `ktfmtCheck`) — project uses Spotless wrapping ktfmt
- Branch protection skipped: private repo on free GitHub plan; document for future enablement
- Added `github-actions` Dependabot ecosystem (beyond original plan scope) for Actions version currency

**Files created:**
- `.github/workflows/ci.yml`
- `.github/dependabot.yml`

---

### Phase 6: Docker & Release Pipeline

**Goal:** Docker image build and push to GHCR on main merge, plus versioned releases triggered by git tags.

**Status:** Complete (2026-03-14) — 2/2 plans executed

**Success Criteria:**
1. [x] Merging to main builds a Docker image and pushes it to GHCR tagged `latest`
2. [x] Pushing a `v*.*.*` git tag builds a Docker image and pushes it to GHCR with the version tag
3. [x] Pushing a version tag creates a GitHub Release with auto-generated changelog
4. [x] GHCR images publicly pullable (manual visibility toggle required after first push)

**Plans:**
- [x] **06-01** — Create cd.yml CD workflow (Docker build/push to GHCR + GitHub Release)
- [x] **06-02** — End-to-end verification: push v1.1.0 tag and confirm GHCR + GitHub Release

**Key decisions:**
- `workflow_run` trigger — CD chains after CI, never runs unless CI passes
- `docker/metadata-action@v5` for tag computation — avoids manual shell tag logic
- Multi-platform build (amd64 + arm64) via QEMU + Docker Buildx
- `softprops/action-gh-release@v2` for GitHub Release creation
- GHA layer caching (`type=gha,mode=max`) for faster Docker builds
- `release` job `needs: docker` — image published before release page goes live

**Files created:**
- `.github/workflows/cd.yml`
